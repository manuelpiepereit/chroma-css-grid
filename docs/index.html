ChromaCSS

A highly versatile css grid framework for simple but customizable web development.

@version: 2.5.0

Why another grid framework? Well, because I was kind of unhappy with existing grid systems, simply wanted to dive deeper into Sass and yada yada yada....
Fuck it, it's doing what it should do for a couple of years now.

simple & minimal

Like bigger Frameworks you can simply include the css file and are ready to go. Or switch off the default scaffolding and use just those SASS mixins you really need.

modular & customizable

Import only those partials you need and customize the grid to your own needs. With an individual column amount, maximum site width and custom media query breakpoints.

BEM & SASS

We're using BEM-Syntax where needed or optionally also for row and columns.
And everything is done with Sass.

getting started

these docs are not at great length, but should give you an idea about what you are dealing with.

requirements

At least some knowledge about html and css. But the following things would be really good.

  1. Bower
  2. Gulp/Grunt/Webpack...

install

  1. download/clone this repository
  2. install bower package bower install
  3. decide how you want to use ChromaCSS

kickstart

The simplest way to use this framework is to just include the minified CSS file and start working.

<link rel="stylesheet" href="{path-to-chroma-css}/css/chroma.min.css">

For a more profound experience and to customize the grid system you first need to set your own settings and then simply import all SCSS files at once.

@import 'your-custom-settings';
@import '{path-to-chroma-css}/scss/all';

grid _layout.grid.base.scss

So, if you want to, you also can import only those files you really need. But as this is the base of the ChromaCSS grid system, you'll at least need these three imports:

import & settings

@import 'grid.settings';
@import '[path_to_bower_components]/sass-mq/mq';
@import 'layout.grid.base';

You always have to define your own grid settings before importing the grid.settings file.
Take a look at all the settings you can change, to customize ChromaCSS to fit perfectly for you.

// settings
$chroma-grid-amount           : 12;     // number of total columns
$chroma-grid-gutter           : 10px;   // gutter between columns
$chroma-grid-max-width        : null;   // maximum container width in px

$chroma-is-static-layout      : false;  // static container layout with fixed width

$chroma-grid-scaffold-default : true; // creates default grid classes

usage & mixins

css selector naming scheme


.cols-{column}               // where {column} is the span you'd like
.cols-{column}[/fraction]    // by default the most important (1/2, 1/3, 1/4, 1/1) fractions are created

mixins


@include chroma-container([$max-width])    // defaults to $chroma-grid-max-width.
                                           // set to 'null' if you want full width
@include chroma-row()                      // creates a row element

@include chroma-column()                   // the generic column class
@include chroma-columns($column-amount, {$total-columns, $breakpoint}); // creates column

@include chroma-clearfix()                 // the usual suspect

usage & mixins

css selector naming scheme

.cols-{column}@{breakpoint}

.cols-{column}--{modifier}@{breakpoint}
<div class="cols-12 cols-6@md cols-3@lg">

<div class="cols-12 cols-9@lg cols-3--push@lg">

mixins

We are using a wrapper mixin for sass-mq. It just takes one key of your breakpoints and uses the $chroma-is-mobile-first variable to determine whether to use $from or $until. Its' just a little bit less code to write.

@include chroma-media($breakpoint) { @content };

// alias for chroma-media($breakoint);
@include at($breakpoint) { @content };

// or use sass-mq
@include sass-mq($from, $until, $and, $media-type) { @content };
						
.sidebar {
  @include at(md) {
    @include chroma-column(3);
    @include chroma-column-right();
    display: block;
  }
}

grid.flexbox _layout.grid.flexbox.scss

import & settings

@import 'layout.grid.flexbox';

$chroma-is-flexbox: false; // if chroma should use flexbox as layout

usage & mixins

css selector naming scheme

// flex columns
.cols-{column}@{breakpoint}         // where {column} is the span you'd like
.cols-x--{ratio}@{breakpoint}       // self-growing column, ratio: 1 to 4

// flex row modifier for alignment
.row--{modifier}@{breakpoint}        // modifier: start | center | end | top | middle | bottom | between | around | stretch | baseline
// flex row modifier for directions
.row--{modifier}@{breakpoint}        // modifier: reverse | column | column-reverse

// flex column modifier
.cols--{modifier}@{breakpoint}       // modifier: bottom | middle | baseline | stretch

// flex column modifier for ordering columns
.cols--{order}@{breakpoint}          // modifier: order-1 | order-2 | order-3 | first | last

mixins

// row alignment 
@include chroma-row-align($align)           // $align: start | center | end | top | middle | bottom | around | between | stretch | baseline
// row directions 
@include chroma-row-direction($direction)   // $direction: reverse | column | column-reverse

// self-growing columns
@include chroma-column-grow($ratio)         // ratio: integer

// column reordering
@include chroma-column-first()
@include chroma-column-last()
@include chroma-column-order($order)        // order: integer

// column alignemnt
@include chroma-column-align($direction)    // directions: top | middle | bottom | stretch | baseline

utils.visualgrid _layout.utils.visualgrid.scss

Shows the grid as page overlay for better and faster page layout development. Toggle html.show-vg to test it.
And shows the currently used media query (bottom left corner): Toggle html.show-mq

import & settings

// import & scaffold
@import 'layout.utils.visualgrid'; 
$chroma-visualgrid-scaffold        : false; // if creating visual grid classes

// settings
$chroma-visualgrid-show-edges      : false; // is showing 1px edge lines
$chroma-visualgrid-show-baseline   : true; // is showing 1px baselines

$chroma-visualgrid-color           : rgba(#ff7300, 0.1); / columns color
$chroma-visualgrid-zindex          : 9999; // visual grid z index
$chroma-visualgrid-baseline-height : 3rem; // baseline line height
$chroma-visualgrid-baseline-shift  : 0; // baseline shift from top
$chroma-visualgrid-baseline-color  : rgba(#ff7300, 0.2); // baseline color

$chroma-visualgrid-mq-color        : #ff7300 !default; // media query text color

usage & mixins

css selectors

html.show-vg // add 'show-vg' to show the grid
html.show-mq // add 'show-mq' to show the currently used media query

mixins

@include chroma-visualgrid(); // creates visual grid as overlay
@include chroma-visualmq(); // shows media query as pseudo element

utils.debug _debug.grid.scss

Adds colors to all your container, row and column elements during compilation and creates some classes to highlight elements during development.

$chroma-debug-color-row       : rgba(150,150,150,0.05); // background color for rows
$chroma-debug-color-col       : rgba(0,0,150,0.05); // background color for columns
